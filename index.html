SOBRE MI PROYECTO

¿Cómo funciona WebSec Audit?

¿Por qué creé esta herramienta?

Desarrollé esta plataforma como una solución "all-in-one" para ciberseguridad. Mi objetivo fue combinar el análisis dinámico (DAST) con el análisis estático (SAST) local en una interfaz moderna y rápida. Quise crear algo que identifique vectores de ataque de forma sencilla para cualquier desarrollador.

¿En qué me baso para el puntaje?

He diseñado un sistema Deductivo de Salud inspirado en el estándar industrial CVSS. El sitio comienza con 100 puntos y voy restando según la gravedad de lo que encuentro:

  • Crítico (-15): Fallos que comprometen todo (SQLi, Secretos, .env).
  • Alto (-8): Riesgos severos (WordPress vulnerable, sin HSTS).

Mi sistema de detección de Puertos

Para lograr precisión, implementé un Proxy de Bajo Nivel en PHP que me permite realizar conexiones TCP reales. Así puedo detectar puertos abiertos como el 3306 (DB) o 21 (FTP) y leer sus "banners" para saber exactamente qué software estás corriendo.

¿Cómo identifico las Cabeceras?

Analizo las respuestas HTTP de tu servidor buscando escudos de protección. Me enfoco en:
• CSP: Para que nadie inyecte scripts (XSS).
• HSTS: Para obligar el uso de HTTPS.
• X-Frame-Options: Para evitar el secuestro de clics (Clickjacking).
